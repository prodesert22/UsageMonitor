# v0.4.0

Adds multi-account support: every provider can now track several logins or keys
side by side.

## Changes

- New per-provider `account` commands: `add`, `remove`, `list`, `set`, `unset`,
  `enable`, `disable`, `auto`. Each account carries its own credentials, an
  optional label, and an enable toggle.
- `fetch` emits one block per enabled account, headed by the account label
  (falling back to `provider (name)`); `fetch <provider> --account <name>`
  restricts to a single account.
- The auto-detected `default` account is fetched **alongside** named accounts
  rather than being replaced. Disable it with
  `<provider> account disable default` (an explicit `default` account also takes
  over that slot).
- `opencode-go workspace add|remove|list` take `--account <name>` (default
  `default`); workspaces are now tracked per account.
- The bare `<provider> set`/`unset` commands operate on the `default` account.
- `<provider> show` lists the provider state and every configured account, plus
  the auto-detected default when applicable.
- Config layout is now nested per account
  (`[providers.<id>.accounts.<name>]`); each `UsageSnapshot` carries its
  `account_id`/`account_label`.
- New provider documentation under `docs/providers/` for `claude`, `codex`,
  `anthropic`, and `openai`, with a per-provider index. The OAuth providers
  (`claude`, `codex`) document why copying a credentials file fails (token
  rotation) and how to set up isolated logins per account.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli claude account add work --label "Work"
./target/release/usage-monitor-cli fetch
```
