# v0.1.3

Adds the provider enable/disable system, mirroring CodexBar's provider toggles.

## Changes

- Providers can be explicitly enabled or disabled, persisted in
  `~/.config/usage-monitor/config.toml`. Without an explicit toggle, a
  provider is auto-enabled when its credentials are detected on the machine
  (credential files for `claude`/`codex`, API key env vars for
  `anthropic`/`openai`).
- New CLI subcommands: `enable <provider>`, `disable <provider>`, and
  `auto <provider>` (returns to auto-detection). `list` now shows the
  resolved state of each provider.
- `fetch` without a provider fetches all enabled providers. Explicitly
  fetching a disabled provider is blocked with a clear error.
- Friendlier rate-limited error message.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli list
```
