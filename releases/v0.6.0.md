# v0.6.0

Adds desktop widgets for KDE Plasma 6 and Waybar.

## Added

- `usage-monitor-cli widget waybar` prints a Waybar custom-module JSON payload on
  one line; `usage-monitor-cli widget kde` prints the same payload for the KDE
  helper (`--pretty` for inspection).
- KDE Plasma 6 plasmoid in `widgets/kde/package`, a faithful port of the
  `codexbar-kde` plasmoid:
  - Usage popup with one card per provider/account (Session/Weekly/Monthly bars,
    reset times, stale/error states) and a pin button to keep it open.
  - Native KDE config window (right-click → Configure, or the popup Settings
    button) split into **General**, **Providers**, and **Order** pages, plus the
    native **About** page. Display preferences use Apply/OK; provider toggles,
    account/workspace management, and cache clear apply immediately.
  - Add/remove named accounts per provider with a form shaped to each provider's
    auth type, and add/remove opencode-go workspaces.
  - The Python helper finds `usage-monitor-cli` (env override or `~/.cargo/bin`,
    `/usr/bin`, …), refreshes data, and caches the last good result under
    `$XDG_CACHE_HOME/usage-monitor-kde/last.json`.
- Account emails now appear for each account: Codex decodes its email from the
  OAuth `id_token`, shown in the CLI text output (`Account:`), the widget JSON
  (`account_email`), and KDE cards (falling back to the plan when no email is
  available, e.g. Claude).
- Waybar wrapper in `widgets/waybar/usage-monitor-waybar`.
- Tests for the Rust widget payload + Codex email and the Python helper/wrapper.
- Widget documentation under [`docs/widgets/`](../docs/widgets/README.md) and an
  Installation troubleshooting section.

## Changed

- Refactored the CLI crate into smaller modules (`cli`, `commands`, `dynamic`,
  `fetch`, `output`, and `widget`); the binary now identifies itself as
  `usage-monitor-cli` consistently.
- Added a local quality gate (`python scripts/check_quality.py`) covering
  rustfmt, Clippy, `cargo test`, Python lint/tests, and QML linting.

## Build from source

```bash
cargo build --release
# binary at target/release/usage-monitor-cli
```
