# v0.5.0

Adds thirteen new native Linux fetchers, ported from the CodexBar macOS
implementation, bringing the total to 26 providers with real usage fetching.

## Changes

- New native Linux fetchers:
  - `cursor` — plan percentage + on-demand spend via the web dashboard
    (`/api/usage-summary`, legacy `/api/usage`), browser session cookie auth.
  - `copilot` — premium-interaction and chat quotas via
    `api.github.com/copilot_internal/user`, GitHub OAuth/PAT token auth.
  - `perplexity` — recurring/bonus/purchased credits via `/rest/billing/credits`
    with waterfall attribution, browser session cookie auth.
  - `gemini` — Gemini Code Assist per-model daily quotas via Cloud Code
    (`loadCodeAssist` + `retrieveUserQuota`), reusing gemini-cli OAuth with
    automatic token refresh.
  - `antigravity` — Antigravity Code Assist per-model daily quotas via Cloud
    Code (`fetchAvailableModels` + `retrieveUserQuota`), Google OAuth auth.
  - `abacus` — organization compute points via `_getOrganizationComputePoints`
    + `_getBillingInfo`, browser session cookie auth.
  - `devin` — daily/weekly quota via `/api/<org>/billing/quota/usage`, Bearer
    token + organization auth.
  - `kimi` — coding weekly quota + rate limit via the kimi.com `GetUsages`
    billing RPC, kimi-auth token.
  - `kimik2` — credits via the kimi-k2.ai credits endpoint, API-key auth.
  - `minimax` — coding/token-plan quota via the `remains` endpoints, API-key
    auth (interval + weekly windows).
  - `mistral` — month-to-date API spend via `admin.mistral.ai` billing, browser
    session cookie auth.
  - `ollama` — cloud session/weekly usage scraped from `ollama.com/settings`,
    browser session cookie auth.
  - `zai` — coding-plan token/prompt quotas via the z.ai quota API, API-key auth.
- Browser-cookie providers take the session cookie/token from config (or an env
  var) rather than auto-importing it from a browser, which is macOS-specific.
- Adds dynamic provider config subcommands so a provider ID does not require a
  dedicated clap enum variant, with `help`/`-h`/`--help` (and a bare
  `<provider>` / `<provider> account`) printing contextual usage. Help flags are
  intercepted before parsing, so `<provider> account add -h` no longer creates a
  junk account.
- New per-provider docs under `docs/providers/` for `cursor`, `copilot`,
  `perplexity`, `gemini`, `antigravity`, `abacus`, `devin`, `kimi`, `kimik2`,
  `minimax`, `mistral`, `ollama`, and `zai`.

## Build from source

```bash
cargo build --release
# binary at target/release/usage-monitor-cli
```
