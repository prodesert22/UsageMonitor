# v0.3.1

Improves multi-provider fetch performance by running enabled providers in
parallel.

## Changes

- `usage-monitor-cli fetch` without a provider now fetches all enabled
  providers concurrently instead of one after another.
- Output order remains deterministic because results are still printed in the
  sorted provider-id order returned by the registry.
- `ProviderRegistry::fetch_enabled` and `ProviderRegistry::fetch_all` now poll
  provider fetch futures concurrently.
- Added a registry test that verifies enabled providers are fetched in parallel.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli fetch
```
