# v0.2.0

Adds the OpenCode Go provider with multi-workspace support, provider config
commands, and a code-quality pass across the workspace.

## Changes

- New `opencode-go` provider: tracks OpenCode Go workspace usage (rolling
  5h, weekly, and optional monthly windows) by scraping the opencode.ai
  dashboard with a manual session token. See
  [docs/providers/opencode-go.md](../docs/providers/opencode-go.md).
- Multiple workspaces are supported with a single token: the first fills the
  primary windows, the rest become named extra windows.
- Workspace names: fetched automatically from the discovery payload, or set
  manually (`opencode-go workspace add <id> "<name>"`, which takes
  precedence). Window labels use the name instead of the raw `wrk_...` id.
- New CLI commands:
  - `config set/get/unset <provider> <key> [value]` — manage provider config
    values (e.g. the opencode-go `token`); secret values are masked on show.
  - `opencode-go workspace add/remove/list` — manage tracked workspaces;
    accepts a `wrk_...` id or a dashboard URL.
- Provider config now persists flat under `[providers.<id>]` in
  `~/.config/usage-monitor/config.toml`, with `token` and `workspaces`
  (a TOML array) as typed fields.
- Code quality: resolved all Clippy warnings (Rust 2024 let-chains),
  formatted with rustfmt, added crate-level docs, and expanded test coverage
  (provider detection helpers, error messages, CLI end-to-end tests).

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli list
```
