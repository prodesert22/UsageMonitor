# v0.7.0

Adds a built-in widget installer and folds the desktop widgets into the
`usage-monitor-cli` crate so a single `cargo install` ships everything.

## Added

- `usage-monitor-cli widget install <kde|waybar|all>` installs the embedded
  widgets: the KDE plasmoid via `kpackagetool6` (with a `--force` direct-copy
  fallback) and the Waybar wrapper symlinked into `~/.local/bin`.
- `usage-monitor-cli widget uninstall <kde|waybar|all>` removes an installed
  widget, and `usage-monitor-cli widget doctor` prints the resolved install
  paths and tool availability.
- Automatic widget upgrades: `widget install` records the installed version and
  adds a login autostart entry that runs the new `usage-monitor-cli widget sync`,
  which reinstalls any installed widget older than the CLI. Upgrading the binary
  propagates to the desktop widgets on the next login without a manual reinstall.

## Changed

- The former `usage-monitor-core` crate was merged into `usage-monitor-cli` as
  internal library modules, so crates.io publishing only needs one package.
- Widget sources moved into the CLI asset tree
  (`usage-monitor-cli/assets/{kde,waybar}`) and are embedded in the binary; the
  `widgets/` directory keeps only the Python unit tests, which load the helpers
  from the asset tree.
- Workspace crate version and KDE plasmoid metadata were bumped to `0.7.0`.

## Fixed

- The widget installer no longer copies `__pycache__`/`.pyc` files into the
  installed KDE plasmoid or Waybar staging directory.
- The KDE panel bar now shows the bundled Usage Monitor logo and scales it to the
  panel thickness, so the icon is no longer clipped (and hidden) on thin panels.

## Install

This is the first release published to
[crates.io](https://crates.io/crates/usage-monitor-cli):

```bash
cargo install usage-monitor-cli
# or from a local checkout: cargo install --path usage-monitor-cli

# then install a desktop widget (optional):
usage-monitor-cli widget install all
```

## Build from source

```bash
cargo build --release
# binary at target/release/usage-monitor-cli
```
