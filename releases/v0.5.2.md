# v0.5.2

Improves the human-readable (non-`--json`) output: timestamps now render in your
system's local timezone with relative, friendlier reset descriptions.

## Changes

- `Collected at` is shown in the system-local timezone with the UTC offset, e.g.
  `00:16 14/06/2026 (UTC-03:00)`, instead of a raw UTC timestamp.
- Rate-window reset times are now relative and local:
  - today → `resets at HH:MM`
  - tomorrow → `resets tomorrow at HH:MM`
  - otherwise → `resets <Weekday> dd/mm at HH:MM` (the year is shown only when it
    differs from the current one).
- The `--json` output is unchanged: it keeps the raw UTC RFC 3339 timestamps for
  machine consumption.

## Build from source

```bash
cargo build --release
# binary at target/release/usage-monitor-cli
```
