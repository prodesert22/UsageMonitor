# v0.3.3

Hardens OpenCode Go workspace configuration validation.

## Changes

- `opencode-go workspace add` now rejects unchanged duplicates instead of
  silently succeeding.
- Re-adding an existing workspace with a different display name still updates
  that name.
- `opencode-go workspace remove` now errors when the workspace is not
  configured.
- Workspace display names with commas are rejected because workspace config is
  serialized as comma-separated entries for provider fetch context.
- Manually duplicated workspace entries in `config.toml` are canonicalized on
  the next `workspace add` or `workspace remove` operation.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli opencode-go workspace list
```
