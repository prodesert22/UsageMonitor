# v0.5.1

Adds the two providers that speak protobuf instead of JSON, bringing the total
to 28 native Linux fetchers.

## Changes

- New native Linux fetchers:
  - `grok` — Grok credit usage via the `GetGrokCreditsConfig` gRPC-Web billing
    RPC. The protobuf response is generically scanned for the usage percentage
    and reset timestamp (the schema is not published). Bearer-token or
    browser-cookie auth.
  - `windsurf` — Windsurf daily/weekly quota via the `GetPlanStatus` Connect RPC
    (`exa.seat_management_pb.SeatManagementService`). The raw-protobuf request and
    response are hand-encoded/decoded from CodexBar's reverse-engineered field
    numbers. Devin-style session-token auth.
- New `provider::proto` module: a minimal protobuf wire reader/encoder shared by
  the two protobuf providers (varint, length-delimited, fixed32/64), with bounds
  checks so malformed input yields errors rather than panics.
- New per-provider docs under `docs/providers/` for `grok` and `windsurf`.

## Build from source

```bash
cargo build --release
# binary at target/release/usage-monitor-cli
```
