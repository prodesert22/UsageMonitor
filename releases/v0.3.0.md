# v0.3.0

Standardizes the CLI around provider-first configuration commands and makes
OpenCode Go authentication easier to set up reliably from browser cookies.

## Changes

- Provider config commands moved from the generic `config` namespace to each
  provider namespace:
  - `opencode-go set token <cookie>` replaces
    `config set opencode-go token <cookie>`.
  - `opencode-go show` replaces `config get opencode-go`.
  - `opencode-go unset token` replaces `config unset opencode-go token`.
  - The same `show`/`set`/`unset` pattern is available for `claude`, `codex`,
    `anthropic`, and `openai`.
- `opencode-go workspace add/remove/list` remains under the `opencode-go`
  provider because workspaces are OpenCode Go-specific config, not a global
  concept.
- OpenCode Go token handling now accepts multiple input formats:
  - bare `Fe26...` auth cookie values, automatically sent as `auth=<value>`;
  - explicit `auth=Fe26...` cookie pairs;
  - full browser `Cookie` header values;
  - copied header lines starting with `Cookie:`.
- Documentation now clearly recommends single quotes around cookie values to
  prevent shell truncation or expansion of characters like `;`, `$`, `*`, and
  spaces.
- CLI integration tests were updated for the provider-first commands, and
  OpenCode Go cookie normalization tests were added.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli list
```
