# v0.3.2

Improves the OpenCode Go terminal output when multiple workspaces are fetched.

## Changes

- Normal text output now renders provider and OpenCode Go workspace headers as
  framed text blocks: a line of `_` characters, the provider/workspace name,
  then a continuous box-drawing line using `─` characters. The lines span the
  dynamically calculated visible width of the current output block.
- `usage-monitor-cli fetch opencode-go` now groups windows by workspace under
  those workspace headers.
- Workspace window labels are printed without repeating the workspace prefix
  under each header (`Rolling (5h)`, `Weekly`, `Monthly`).
- OpenCode Go monthly usage parsing now skips scalar billing fields such as
  `monthlyUsage:null` and uses the real workspace `monthlyUsage` object with
  `usagePercent`.
- `usage-monitor-cli fetch` also shows the same workspace grouping when
  `opencode-go` is included alongside other enabled providers.
- JSON output is unchanged.

## Build from source

```bash
cargo build --release
./target/release/usage-monitor-cli fetch opencode-go
```
